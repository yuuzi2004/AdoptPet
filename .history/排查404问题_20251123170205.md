# 排查 404 错误详细步骤

## 当前配置检查清单

### 1. 确认访问路径
- 正确的访问路径应该是：`http://localhost:8080/Pet1.0/pet/list`
- 注意：`Pet1.0` 是项目的 context path（在 pom.xml 的 finalName 中定义）

### 2. 检查项目是否正确部署

**方法 A：检查 Tomcat webapps 目录**
- 找到 Tomcat 安装目录（例如：`E:\apache-tomcat-10.1.45\webapps`）
- 确认 `Pet1.0` 文件夹存在
- 确认 `Pet1.0/WEB-INF/classes/com/pet/adopt/controller/PetListServlet.class` 存在

**方法 B：检查 Tomcat 日志**
- 查看 `catalina.out` 或控制台日志
- 查找是否有部署成功的消息
- 查找是否有错误信息

### 3. 测试步骤

**步骤 1：测试 JSP 是否部署**
访问：`http://localhost:8080/Pet1.0/list.jsp`
- 如果这个能访问，说明项目已部署
- 如果这个也 404，说明项目没有正确部署

**步骤 2：测试根路径**
访问：`http://localhost:8080/Pet1.0/`
- 应该会重定向到 `/pet/list`

**步骤 3：测试 Servlet**
访问：`http://localhost:8080/Pet1.0/pet/list`
- 如果前两步都正常，这一步应该也能工作

### 4. 常见问题

**问题 1：项目没有部署**
- 解决：在 IDE 中重新部署项目
- 或者手动复制 `target/Pet1.0.war` 到 `webapps` 目录

**问题 2：Tomcat 版本不匹配**
- Tomcat 10 需要 `jakarta.*`
- Tomcat 9 需要 `javax.*`
- 当前配置是 `javax.*`，需要 Tomcat 9

**问题 3：编译错误**
- 检查是否有编译错误
- 运行 `mvn clean compile` 查看错误

### 5. 快速修复步骤

1. **完全清理**
   ```bash
   mvn clean
   ```

2. **重新编译**
   ```bash
   mvn compile
   ```

3. **检查编译是否成功**
   - 确认 `target/classes/com/pet/adopt/controller/PetListServlet.class` 存在

4. **重新部署**
   - 在 IDE 中停止 Tomcat
   - 删除 `webapps/Pet1.0` 文件夹
   - 重新部署项目
   - 启动 Tomcat

5. **查看日志**
   - 检查 Tomcat 启动日志
   - 查找是否有 Servlet 注册信息
