# 解决 404 错误：/pet/list 不可用

## 问题
访问 `/pet/list` 时出现 404 错误，说明 Servlet 没有被正确部署或识别。

## 可能的原因

1. **项目没有重新编译和部署**
2. **Tomcat 版本不匹配**（如果使用 Tomcat 10，javax 版本不兼容）
3. **Servlet 注解没有被识别**

## 解决步骤

### 步骤 1：完全重新编译和部署

1. **停止 Tomcat**

2. **清理项目**
   ```bash
   mvn clean
   ```

3. **重新编译**
   ```bash
   mvn compile
   ```

4. **打包 WAR**
   ```bash
   mvn package
   ```

5. **在 IDE 中重新部署**
   - 删除旧的部署
   - 重新部署项目

6. **启动 Tomcat**

### 步骤 2：检查 Tomcat 版本

**重要：** 如果使用 **Tomcat 10**，javax 版本不兼容！

- **Tomcat 10** → 需要使用 `jakarta.*`（我们刚改回 javax）
- **Tomcat 9** → 可以使用 `javax.*`

**解决方案：**
- 选项 A：切换到 Tomcat 9
- 选项 B：改回使用 jakarta（如果必须用 Tomcat 10）

### 步骤 3：验证部署

1. **检查 Tomcat 日志**
   - 查看是否有 Servlet 部署成功的日志
   - 查找类似：`Servlet [PetListServlet] mapped to [/pet/list]`

2. **检查部署目录**
   - 确认 `webapps/Pet1.0/WEB-INF/classes/com/pet/adopt/controller/PetListServlet.class` 存在

3. **直接访问测试**
   - 先访问：`http://localhost:8080/Pet1.0/list.jsp`（测试 JSP）
   - 再访问：`http://localhost:8080/Pet1.0/pet/list`（测试 Servlet）

### 步骤 4：如果还是不行，在 web.xml 中显式配置

如果注解不生效，可以在 web.xml 中添加显式配置。
