# 解决 IDE 无法识别 JSTL 标签的问题

## 问题
IDE 报错：`无法解析符号 'c:forEach'`，无法识别 JSTL 标签

## 解决方案

### 方法一：刷新 Maven 项目（推荐）

1. **在 IntelliJ IDEA 中：**
   - 右键点击 `pom.xml`
   - 选择 `Maven` → `Reload Project`
   - 或者打开 Maven 工具窗口，点击刷新按钮

2. **清理并重新导入：**
   - `File` → `Invalidate Caches / Restart`
   - 选择 `Invalidate and Restart`

### 方法二：确保依赖正确

1. **检查依赖是否下载：**
   - 打开 Maven 工具窗口
   - 展开 `Dependencies`
   - 确认 `jakarta.servlet.jsp.jstl` 和 `jakarta.servlet.jsp.jstl-api` 存在

2. **如果依赖缺失，执行：**
   ```bash
   mvn clean install
   ```

### 方法三：配置 IDE 的 JSP 支持

1. **File → Settings → Languages & Frameworks → JSP**
   - 确保启用了 JSP 支持
   - 检查 JSP 版本设置

2. **File → Project Structure → Modules**
   - 确认 `src/main/webapp` 被标记为 Web Resource Directory

### 方法四：如果 IDE 仍然报错

**重要：** IDE 的报错可能只是警告，不影响运行时！

1. **验证运行时是否正常：**
   - 重新编译：`mvn clean compile`
   - 重新部署到 Tomcat
   - 访问：`http://localhost:8080/Pet1.0/pet/list`
   - 如果页面正常显示，说明代码没问题，只是 IDE 的识别问题

2. **忽略 IDE 警告：**
   - 如果运行时正常，可以暂时忽略 IDE 的红色波浪线
   - 或者使用 `@SuppressWarnings` 注释（JSP 中不适用）

## 当前配置

- **JSTL URI:** `jakarta.tags.core`（Jakarta JSTL 2.0 标准）
- **依赖版本:** 2.0.0
- **web.xml 配置:** 已添加标签库映射

## 验证

运行项目后，如果页面能正常显示宠物列表，说明 JSTL 工作正常，IDE 的报错只是识别问题。
