# 修复 MySQL 密码问题

## 问题原因
错误信息：`Access denied for user 'root'@'localhost' (using password: YES)`

这说明配置文件中的密码 `HE04140626` **不正确**。

## 解决方案

### 方法一：找到正确的密码（推荐）

1. **使用 Navicat 测试连接**
   - 打开 Navicat
   - 尝试不同的密码（常见密码：空密码、root、123456、你设置的密码等）
   - 找到能成功连接的密码

2. **检查 MySQL 配置文件**
   - 查看 MySQL 安装目录下的 `my.ini` 或 `my.cnf`
   - 但密码通常不会明文存储

3. **询问安装 MySQL 时设置的密码**
   - 如果是你自己安装的，回忆一下安装时设置的 root 密码

### 方法二：重置 MySQL root 密码

如果找不到密码，可以重置：

#### Windows 重置密码步骤：

1. **停止 MySQL 服务**
   ```cmd
   net stop MySQL
   ```
   （服务名可能是 MySQL80、MySQL57 等，在"服务"中查看）

2. **以跳过权限表方式启动 MySQL**
   ```cmd
   mysqld --skip-grant-tables
   ```
   保持这个窗口打开

3. **打开新的命令窗口，登录 MySQL（无需密码）**
   ```cmd
   mysql -u root
   ```

4. **执行重置密码命令**
   ```sql
   USE mysql;
   UPDATE user SET authentication_string=PASSWORD('新密码') WHERE User='root';
   FLUSH PRIVILEGES;
   EXIT;
   ```
   
   对于 MySQL 8.0+，使用：
   ```sql
   USE mysql;
   ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
   FLUSH PRIVILEGES;
   EXIT;
   ```

5. **关闭跳过权限表的 MySQL 窗口，正常启动 MySQL 服务**
   ```cmd
   net start MySQL
   ```

### 方法三：使用空密码（如果允许）

如果 MySQL 允许空密码连接，可以尝试：

1. 修改 `db.properties`：
   ```properties
   jdbc.password=
   ```

2. 或者删除密码行（不推荐）

## 找到正确密码后

1. **修改配置文件**
   - 打开 `src/main/resources/db.properties`
   - 修改 `jdbc.password=你的正确密码`

2. **重新编译和部署**
   ```bash
   mvn clean compile
   ```

3. **重启 Tomcat**

4. **测试连接**
